# Проект спринта "Организация Data Lake"

Вот обновленный файл в твоем формате. Я добавил технические детали, которые мы реализовали (про 27 дней, формулу и структуру DAG), сохранив твою структуру и лаконичность.

Проектная работа: Организация Data Lake для социальной сети
Описание проекта
Цель работы — переработка структуры Data Lake для внедрения геоаналитики и функционала рекомендаций друзей в Австралии. Проект включает в себя обработку событий пользователей, сопоставление их с географическими координатами городов и расчет метрик активности.

Структура хранилища
1. Источники данных (Raw Layer)
События пользователей: /user/master/data/geo/events

Формат: Parquet.

Содержит: сообщения, подписки, реакции, регистрации. Включает поля lat и lon для исходящих сообщений.

Справочник городов: /user/s26586465/data/geo/geo.csv

Формат: CSV (разделитель ;).

Содержит: координаты центров городов Австралии и их часовые пояса.

2. Целевые витрины (Analytics Layer)
Все витрины сохраняются в формате Parquet по пути /user/s26586465/analytics/:

user_mart: Гео-профиль пользователя (актуальный адрес, домашний адрес, количество путешествий, список городов, местное время).

zone_mart: Гео-аналитика по зонам (количество событий разных типов в разрезе недель и месяцев).

recommendation_mart: Рекомендации друзей на основе геопозиции (дистанция < 1 км), общих каналов и отсутствия прошлых контактов (через Direct Messages).

Логика обработки данных
Определение местоположения
Для каждого события с координатами вычисляется расстояние до всех городов из справочника geo.csv по формуле гаверсинуса. Событие привязывается к городу с минимальным расстоянием.

Домашний адрес (Home City)
Домашним адресом считается город, в котором пользователь находился дольше 27 дней подряд. В коде реализована проверка непрерывности пребывания через оконные функции и анализ последовательности посещений.

Автоматизация
Обновление витрин автоматизировано с помощью Airflow. DAG настроен на ежедневный запуск (@daily). Пайплайн включает последовательный запуск трех Spark-задач:

Расчет профилей пользователей (user_mart).

Расчет агрегатов по зонам и активности (geo_zones).

Генерация рекомендаций (friend_recommendation).

Технологический стек
Язык: Python (PySpark).

Инфраструктура: Hadoop HDFS, Spark Cluster (Yandex Cloud Dataproc).

Оркестрация: Apache Airflow (SparkSubmitOperator).
